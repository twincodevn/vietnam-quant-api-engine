# Python Backend Guidelines

- **Version:** Python 3.10+
- **Typing:** Strict Type Hinting is mandatory for all functions and methods.
- **FastAPI & Data Validation:** Use Pydantic models for request/response validation and data structuring.
- **Code Style:** Strictly adhere to PEP 8 standards. Use clear, descriptive naming conventions.
- **Modularity:** Keep the logic cleanly separated (e.g., `decision_engine` for quant logic, `main` for routing and API endpoints).
